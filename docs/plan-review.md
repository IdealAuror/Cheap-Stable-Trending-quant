# epic-leek-quant 项目计划审查报告

> 审查对象：三份原文档
> - `epic-leek-value-investing.md`（理论 + 聚宽回测框架，已提交）
> - `docs/agent-workflow.md`（Agent 工作流程 / 执行规范，未提交）
> - `research/thinking-prompt.md`（专业量化研究员分析框架，未提交）
>
> 审查目的：定位文档间口径冲突、代码缺陷与结构性缺口，为统一执行计划（`PROJECT-PLAN.md`）与 DeepSeek 提示词模板提供依据。

> 审查原则：**以 `thinking-prompt.md` 的严格学术标准为准绳，以 theory 文档的乐观数字为"待验证假设"。** 本报告不修改原文档，仅记录问题与修复方向。

## 一、逐份审查

### 1.1 `epic-leek-value-investing.md`（理论 + 框架）

| 维度 | 评价 |
|------|------|
| **优点** | 投资哲学清晰、案例坦诚（含失利）、五大原则到五大因子的映射逻辑链完整；第四章给出端到端可读的聚宽参考代码，入门价值高。 |
| **薄弱点** | ① 结论偏乐观，"年化 16–20%、超额 409%、显著且持续"等数字未标注样本/区间/口径，易被误读为已验证结论；② 第四章代码有多处 bug（见第三节）；③ 完全未处理退市股、涨跌停/T+1/停牌，回测结果系统性偏乐观；④ 二值因子做 Z-score 属冗余变换。 |
| **与其他文档冲突** | 调仓频率（月频）与 `thinking-prompt.md` 推导的季度更优冲突；成本模型（`PerTrade + 0.002 滑点`）远比 `thinking-prompt.md` 要求的保守值乐观；基调与 `thinking-prompt.md` 的逐条质疑对立。 |

### 1.2 `docs/agent-workflow.md`（Agent 工作流程）

| 维度 | 评价 |
|------|------|
| **优点** | 工程纪律强：Senior→spec→Flash→**代码审核（看结果前）**→结果判定的管线设计专业；预注册通过标准、交互式澄清通道、迭代终止规则、确认偏误抑制六机制，这套方法论本身是本项目最有价值的资产之一。 |
| **薄弱点** | ① spec 模板里"换仓频率: 月频"与 `thinking-prompt.md` 的分红税论证不一致；② spec 缺少"退市股处理""涨跌停撮合""停牌冻结"等回测口径字段，导致 Flash 执行时只能自由发挥；③ 澄清通道与迭代轮次**没有落盘机制**——只规定"记录"，未规定记到哪个文件、什么格式；④ "Phase 0 脚手架"缺失，直接进入 Phase 1，没有数据接口封装层。 |
| **与其他文档冲突** | spec 默认"不做市值中性化（仅行业中性化）"，而 `thinking-prompt.md` 强调国企因子极可能是 size/low-beta 代理，需用 Fama-MacBeth 控制市值——两处在方法论的严格度上不一致。 |

### 1.3 `research/thinking-prompt.md`（分析框架）

| 维度 | 评价 |
|------|------|
| **优点** | 本项目学术水准最高的文档。决策门（Gate 1–5）、因子深度追问（壳价值污染、填权概率、回购在 A 股无区分度、国企内生性/代理变量、质量=下行保护）、IC 体系（NW-t、IC_IR 用 NW 标准误、市值加权 IC 对照、多空不对称性）、制度断点表、涨跌停系统性偏差、过拟合防御（Purged CV、DSR、CPCV）、因子拥挤度——几乎涵盖了一个严肃量化研究员会问的全部问题。 |
| **薄弱点** | ① 它是"分析框架/提问清单"，但**没有给出操作结论**——把它的严格标准落到 spec 和执行计划里，正是本审查 + `PROJECT-PLAN.md` 要做的事；② 部分方法（CPCV、Almgren-Chriss）工程成本高，需在计划中排优先级，避免 Phase 1 就被拖垮。 |
| **与其他文档冲突** | 几乎逐条对 theory 文档的乐观结论提出反例（见第二节）。 |

## 二、三份文档间的口径冲突清单

> **判定规则**：冲突时，**以 `thinking-prompt.md` 的严格标准为准绳**；theory 文档的乐观数字降级为"待验证假设"。最终统一口径见 `PROJECT-PLAN.md`。

| # | 议题 | theory 文档现状 | agent-workflow spec | thinking-prompt 要求 | 建议统一口径 |
|---|------|----------------|---------------------|----------------------|-------------|
| C1 | **换仓频率** | 月频（`run_monthly`） | 月频 | 季度更优：月频持股<1月→分红税 20%；季度→10%；且应**对齐财报披露截止日**减少未来函数 | **季度调仓**（每季报披露截止后首个交易日）；月频仅作敏感性对照 |
| C2 | **佣金** | 万 3 买 / 千 1.3 卖 | 未指定 | 万 2.5（资金量大可更低） | **万 2.5 双边** |
| C3 | **印花税** | 含在卖出千 1.3 内 | 未指定 | 千 1（卖），固定 | **千 1（仅卖）**，从千 1.3 中拆出单独列示 |
| C4 | **最低佣金** | `min_cost=5` | 未指定 | 5 元（小资金影响显著） | **5 元/笔** |
| C5 | **滑点** | `PriceSlippage(0.002)` 即 0.2% | 未指定 | 0.2%–0.5%，深度价值策略影响大 | **0.3%**（中等保守） |
| C6 | **冲击成本** | 无 | 未指定 | Almgren-Chriss，按持仓市值/日均成交额分级 | **分级估算**：先按"持仓市值/日均成交额"分档，后续 Phase 4 升级 AC 模型 |
| C7 | **基准** | 沪深 300 全收益（`000300.XSHG`） | 沪深 300 全收益 | 主：沪深 300 全收益；辅：中证 800 全收益；**必须报告跟踪误差/信息比率** | **主沪深 300 全收益 + 辅中证 800 全收益**，强制报告 TE/IR |
| C8 | **退市股** | 未处理（幸存者偏差） | 未指定 | **必须纳入**，退市按跌停排队，停牌期间冻结 | **必须纳入**：退市按实际回报/跌停排队计入；停牌冻结至复牌首日实际成交价 |
| C9 | **涨跌停/T+1** | 未处理 | 未指定 | 禁止"跳过"；需**限价单模拟器**；T+1 下**先卖后买** | 必须实现：涨停排队买入、跌停排队卖出、先卖后买 |
| C10 | **PIT 数据** | `get_fundamentals(date=)`（含修正隐患） | "PIT 模式，不可用期末值" | 必须按 disclosure date；财报修正会覆盖历史 | spec 显式要求校验每个换仓日"实际可得数据"，并在代码审核门禁强检 |
| C11 | **中性化** | 仅口头提及 | 仅行业中性化，**不做市值中性化** | 国企因子极可能是 size/low-beta 代理，需 Fama-MacBeth 控制市值 | 单因子 IC 报告时**同时给出行业中性化 IC 与市值中性化 IC**；Phase 2 用 Fama-MacBeth 筛独立定价能力 |
| C12 | **二值因子处理** | 对 0/1 二值因子做 Z-score | 未指定 | 未明确 | Z-score 对二值因子是冗余线性变换，**不改排序**；二值因子直接等权求和，仅连续因子做 winsorize + Z-score |
| C13 | **基调/结论** | "显著且持续超额收益" | 中立 | 逐条质疑 | theory 数字一律标注为"待验证乐观假设"，不可作为结论 |

## 三、第四章代码缺陷清单

> 以下为 `epic-leek-value-investing.md` 第四章参考代码中定位到的问题。该代码为教学性参考片段，但其中部分缺陷若被复制进真实回测，会直接污染结果。修复后参考片段见 `PROJECT-PLAN.md` 与 `prompts/03-code-review-gate.md`。

| # | 位置 | 问题 | 严重度 | 修复方向 |
|---|------|------|--------|---------|
| B1 | `calculate_all_factors` 中 `median_debt = df['total_liability'] / df['total_assets'].median()` | **逻辑错误**：整列除以一个标量中位数，得到一列无意义值；且 `debt_to_assets` 的计算顺序颠倒（先用了再算） | P0 | 改为 `median_debt = (df['total_liability'] / df['total_assets']).median()`，并先算 `df['debt_to_assets']` 再取中位数 |
| B2 | 对 `factor_*`（0/1 二值）做 Z-score | 冗余线性变换，不改变排序与等权求和结果，徒增复杂度 | P1 | 二值因子直接等权求和；仅连续因子做 winsorize + Z-score |
| B3 | 全程无退市股处理 | 幸存者偏差，系统性高估收益（尤其小市值端） | P0 | 纳入退市股，退市按实际回报/跌停排队计入；回测前明确退市股来源与撮合规则 |
| B4 | 无涨跌停/T+1/停牌处理 | `order_target_percent` 乐观假设全成交；涨跌停/T+1 缺失会系统性高估 | P0 | 实现限价单排队撮合器；调仓顺序先卖后买；停牌冻结至复牌首日 |
| B5 | `get_st_stock_list` 用 `display_name.startswith('ST')` 遍历全市场 | 低效且不严谨（名称变更/退市风险判定不准） | P1 | 改用 `get_extras('is_st', ...)` 或聚宽 ST 标记接口 |
| B6 | `is_new_stock` 用 `datetime.datetime.strptime` 但未 `import datetime` | 运行时 NameError | P1 | 补 import，或统一用 `pd.Timestamp` |
| B7 | `set_commission(PerTrade(buy_cost=0.0003, sell_cost=0.0013, min_cost=5))` | 卖出千 1.3 把印花税揉进佣金，口径不透明；最低佣金在等权小单下影响大但未校准 | P1 | 拆分为佣金 + 印花税 + 滑点三项分别列示（见 C2–C6） |
| B8 | `get_fundamentals(date=current_date)` 的 PIT 语义 | 财报修正会覆盖历史，导致回溯困难/未来函数风险 | P0 | spec 强制要求校验"换仓日实际可得数据"；代码审核门禁必检 |
| B9 | `run_monthly(rebalance, 1)` | 月频与分红税/披露节奏冲突（见 C1） | P0 | 改季度调仓，对齐披露截止日 |
| B10 | `evaluate_performance` 直接读 `get_portfolio_info()` | 未做毛/净收益区分，未报告 TE/IR，未对成本保守性做校验 | P1 | 必须区分毛收益与成本后净收益；强制报告跟踪误差、信息比率 |

## 四、改进优先级

> P0 = 影响结论可信度，必须在 Phase 1 起步前解决；P1 = 影响严谨性，Phase 1–2 内解决；P2 = 增强，Phase 3–4。

### P0（先做）
1. **统一调仓频率为季度**（C1、B9）——分红税、披露节奏、未来函数三重收益。
2. **修复 `debt_to_assets` 计算 bug**（B1）——直接影响因子五的因子值正确性。
3. **退市股纳入 + 涨跌停/T+1/停牌撮合器**（C8、C9、B3、B4）——A 股回测最易低估的误差源，不解决则一切 IC/收益数字均不可信。
4. **PIT 语义强检**（C10、B8）——放进代码审核门禁，看结果前必过。
5. **基调声明**（C13）——所有文档/报告里 theory 的乐观数字必须标注"待验证假设"。

### P1（Phase 1–2 内）
6. 成本模型拆分校准（C2–C6、B7、B10）。
7. 归因与代理变量筛除：用 Fama-MacBeth 检验国企因子是否为 size/low-beta 代理、回购因子是否无效（C11）。
8. 制度断点 + 宏观状态分段统计（借鉴 `thinking-prompt.md` 断点表）。
9. ST 判定改 `get_extras`、补 import、二值因子不做 Z-score（B5、B6、B2）。

### P2（Phase 3–4）
10. Deflated Sharpe Ratio、Combinatorial Purged CV。
11. 因子拥挤度监测、换手-alpha 闭环。
12. 容量估算（千万/亿级冲击成本）、Almgren-Chriss 冲击成本模型。

## 六、总结

三份原文档构成一个**"乐观理论 + 严格方法论 + Agent 工程纪律"**的组合，潜力很高，但存在三类系统性问题：

1. **口径冲突**：调仓频率、成本、中性化、基调在文档间不一致，需以 `thinking-prompt.md` 严格标准统一（见第二节）。
2. **代码缺陷**：第四章参考代码有真实 bug 与多处乐观简化，最严重的（退市/涨跌停/PIT/`debt_to_assets`）会直接污染回测结论（见第三节）。
3. **结构性缺口**：缺 Phase 0 脚手架、落盘机制、DoD、模板（见第五节）。

后续 `PROJECT-PLAN.md` 给出统一口径与阶段计划，`docs/prompts/*` 给出 DeepSeek V4 Pro 的高级行为规范与四步管线提示词，`research/_index.md` 落地研究记录与迭代追溯。
