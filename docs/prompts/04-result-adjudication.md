# Prompt 04 — 结果判定（高级模型，代码审核通过后）

> 本文件是高级模型在**代码审核通过后**分析回测结果、做 Gate 判定的流程。
> 前置条件：`03-code-review-gate.md` 已通过且结论落盘。

---

## 判定流程

```
1. 确认代码审核已通过（读 research/reports/<spec_id>-code-review.md）
   → 通过 → 继续
   → 不通过 → 退回 Flash，不进入结果分析

2. 读 spec 的预注册通过标准（research/specs/<spec_id>.md 的 pass_criteria）
   → 这些标准在 spec 落盘时已锁定，不得调整

3. 读执行偏差记录（research/reports/<spec_id>-deviation.md）
   → 评估每条偏差是否会系统性影响结论方向
   → 若有 P0 偏差（如退市股静默剔除）→ 结果不可信，退回重做

4. 逐条对照通过标准，判定每一条 ✅ / ❌ / ⚠️(不确定)

5. 总判定：
   全部 ✅ → 通过 → 更新 _index.md，进入下一因子 / 下一阶段
   任一 ❌ → 不通过 → 诊断"假设错误 vs 实现错误"
   多个 ⚠️ → 不确定 → 明确不确定性来源，生成新 spec

6. 落盘判定报告到 research/reports/<spec_id>-adjudication.md
```

## 不通过时的强制诊断

不通过**不是终点**，而是诊断的起点。必须区分两种"不通过"：

| 类型 | 含义 | 应对 |
|------|------|------|
| **假设错误（真负）** | 因子在 A 股确实不存在定价能力 | 记录原因，放弃该方向；连续两轮 → 永久放弃 |
| **实现错误（假负）** | 因子可能有效，但本次实现有缺陷 | 修复实现，重跑；不计入"连续两轮"计数 |

诊断维度（逐项排查）：

1. **数据问题**：PIT 模式是否正确？财报修正是否污染？
2. **时间区间**：换一个区间是否翻转？（注意：不可挑区间，仅作诊断）
3. **股票池**：仅在特定样本有效？三样本是否方向一致？
4. **撮合偏差**：退市/涨跌停处理是否过度悲观？
5. **成本过苛**：滑点 0.3% 是否对该因子过重？
6. **因子定义**：连续 vs 二值的口径是否合适？

诊断后产生新的假设 → 生成新 spec（新版本号），记录到 `_index.md`。

## 不确定时的处理

"不确定"指：统计不显著（NW-t 在临界值附近）或效应量太小（IC_IR 在 0.2–0.3 之间）。

- 不得用"根据经验"判定方向；
- 必须明确不确定性来源：样本不足？区间太短？因子衰减？
- 生成新 spec：扩大样本 / 延长回测期 / 子区间分析。

## 判定报告模板

落盘到 `research/reports/<spec_id>-adjudication.md`：

```markdown
# 结果判定 — <spec_id>

## 前置确认
- 代码审核：<spec_id>-code-review.md，结论：通过 ✅
- 审核时未看结果：是 ✅

## 偏差评估
| # | 偏差 | 影响方向 | 严重度 |
|---|------|---------|--------|
| 1 | cash_equivalents 含受限资金 | 高估 EV<0 信号强度 | P1 |

## 逐条判定
| 通过标准 | 结果 | 判定 |
|---------|------|------|
| Rank IC_IR > 0.3 (NW) | 0.34 | ✅ |
| Q1 多头年化 > 基准+2% | 基准+1.5% | ❌ |
| Q1-Q5 单调 | Q3 异常 | ❌ |

## 总判定
- [ ] 通过
- [x] 不通过 — 假设错误
- [ ] 不通过 — 实现错误

1. 三样本不一致 + 市值中性化后失效 → 国企/EV 因子极可能是 size 代理

   （A 股空头不可实现，见 thinking-prompt.md "多空不对称性"）

## 下一步
- 放弃该因子方向？是 / 否
- 若否，新 spec：<spec_id>-v2，待解决：<列出>

## 归因（若通过）
- 超额收益来源：行业配置 / 行业内选股 / 规模暴露 / …
- 与已知风险因子的相关性：size / value / momentum / low-vol / quality
- 残差 alpha 是否显著 > 0
```

## Gate 定义速查

| Gate | 阶段 | 核心规则 |
|------|------|---------|
| Gate 1 | Phase 1 单因子 | IC_IR>0.3(NW) + Q1>基准+2% + Q1-Q5 单调 + 三样本一致 + 行业/市值中性化 IC 显著 |
| Gate 2+3 | Phase 2 合成 | 残差 alpha 显著>0 + 行业中性化后超额不降 + 因子暴露稳定 |
| Gate 4 | Phase 3 压测 | 制度断点/宏观状态不反转 + 参数非边际极端 + DSR 通过多重比较 |
| Gate 5 | Phase 4 实盘 | 成本后净收益为正 + 容量匹配 + 拥挤度未进 90 分位 |

## 抑制确认偏误的六机制（贯穿判定全过程）

1. 代码审核先于结果分析；
2. 通过标准预注册，结果出来后不调整；
3. 不通过时强制诊断"假设错误 vs 实现错误"；
4. 连续两轮 IC_IR<0.2 → 放弃；
5. 所有结论必须有数据支持；
6. 分析前先读 `_index.md` 历史结论。
